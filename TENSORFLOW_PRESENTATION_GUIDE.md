# TensorFlow.js 구현 증명 가이드 (발표용)

## 🎯 발표 시 TensorFlow.js 증명 방법

### 1. **실시간 모니터 시연** (가장 효과적!)

#### 단계별 시연:
1. **초기 상태 (20-30%)**
   - 페이지 처음 로드 시
   - 오른쪽 하단 모니터 클릭하여 확장
   - "사용자 관심도 20-30% (낮음 💤)" 표시

2. **스크롤 시연 (40-50%)**
   - 천천히 아래로 스크롤
   - 관심도가 40-50%로 상승하는 것 확인
   - "보통 📊" 단계로 변경

3. **상호작용 시연 (70%+)**
   - 프로젝트 카드 클릭
   - 스킬 아이콘에 마우스 호버
   - 댓글 작성 시도
   - 관심도가 70% 이상 상승
   - "높음 🔥" 단계로 변경

### 2. **브라우저 콘솔 증명**

#### 콘솔에서 확인할 수 있는 로그:
```
🤖 TensorFlow.js 초기화 중...
✅ TensorFlow.js 버전: 4.22.0
✅ TensorFlow.js 모델 로드 완료
🎯 사용자 관심도: 25.3%
🎯 사용자 관심도: 45.7%
✨ 높은 관심도 감지 - 추천 콘텐츠 제공
```

**발표 시 F12 (개발자 도구) → Console 탭 보여주기**

### 3. **확장된 모니터 정보**

모니터를 클릭하면 다음 정보 표시:
- ✅ **모델 상태**: 활성
- 🔧 **Backend**: webgl (GPU 가속) 또는 cpu
- 📦 **버전**: 4.22.0
- 💾 **메모리 사용량**:
  - Tensors: 실시간 텐서 개수
  - Bytes: 메모리 사용량 (KB)
  - 데이터 버퍼: 활성 버퍼 수

### 4. **발표 PPT에 포함할 스크린샷**

#### 스크린샷 1: 모니터 접힌 상태
- 기본 UI와 함께 오른쪽 하단 모니터 보이는 화면

#### 스크린샷 2: 모니터 확장 상태 (낮은 관심도)
- 관심도 20-30%
- "낮음 💤" 표시
- 모든 상세 정보 보이는 상태

#### 스크린샷 3: 모니터 확장 상태 (높은 관심도)
- 관심도 70%+
- "높음 🔥" 표시
- 메모리 사용량 증가 확인

#### 스크린샷 4: 브라우저 콘솔
- TensorFlow.js 로그 메시지들
- 관심도 변화 로그

### 5. **발표 스크립트 예시**

```
프로젝트를 클릭하고 상호작용하면...

(콘솔 보여주며)
브라우저 콘솔에서도 5초마다 분석 결과가

```

### 6. **기술적 세부사항 (질문 대비)**

#### 모델 구조:
- **입력층**: 5차원 벡터 (스크롤, 클릭, 시간, 호버, 기록)
- **은닉층 1**: 16 units, ReLU
- **은닉층 2**: 8 units, ReLU  
- **출력층**: 1 unit, Sigmoid (0~1 확률)

#### 학습 설정:
- **Optimizer**: Adam (learning rate 0.001)
- **Loss**: Binary Crossentropy
- **Metrics**: Accuracy

#### 실시간 추적:
- **주기**: 5초마다
- **추적 항목**: 
  - 스크롤 깊이 (0~1)
  - 클릭 수 정규화 (0~1)
  - 체류 시간 정규화 (0~1)
  - 호버 수 정규화 (0~1)
  - 행동 기록 누적 (0~1)

### 7. **데모 순서 (실전 팁)**

1. **페이지 새로고침** (F5)
2. **모니터 확장** (클릭)
3. **초기 관심도 확인** (20-30%)
4. **천천히 스크롤** (관심도 상승 확인)
5. **프로젝트/스킬 상호작용** (70%+ 도달)
6. **콘솔 로그 확인** (F12)
7. **메모리 정보 설명** (Tensors, Bytes)

### 8. **예상 질문 & 답변**

**Q: 왜 TensorFlow.js를 사용했나요?**
A: 사용자 행동을 실시간으로 분석하여 관심도를 예측하고, 
향후 개인화된 콘텐츠 추천 시스템으로 확장 가능하기 때문입니다.

**Q: 학습 데이터는 어떻게 수집하나요?**
A: 현재는 실시간 행동 추적만 하고, 추후 수집된 데이터로 
모델을 재학습하여 정확도를 향상시킬 계획입니다.

**Q: 성능 영향은 없나요?**
A: WebGL backend를 사용하여 GPU 가속을 받고, 
5초마다만 분석하여 성능 영향을 최소화했습니다.

## 📸 스크린샷 촬영 가이드

### 촬영 순서:
1. 페이지 로드 → 모니터 접힌 상태 캡처
2. 모니터 클릭 → 확장 상태 (낮은 관심도) 캡처
3. 스크롤 + 클릭 → 확장 상태 (높은 관심도) 캡처
4. F12 → 콘솔 로그 캡처
5. Network 탭 → TensorFlow.js 라이브러리 로드 확인 캡처

### 권장 도구:
- **Windows**: Win + Shift + S
- **Mac**: Cmd + Shift + 4
- **전체 화면**: F11 (브라우저 전체화면)
